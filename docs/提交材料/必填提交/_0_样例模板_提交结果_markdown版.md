## 二、功能说明

### 核心功能清单

系统核心能力一览：

- 多角色 Agent 协作：信息采集、分析师、报告撰写、质检四大 Agent 独立运行、协同工作

- 结构化知识抽取：定义竞品知识 Schema（功能树、定价模型、用户画像），确保输出一致性

- DAG 任务流转：Agent 间通过 function calling 结构化消息传递，支持质检打回与迭代闭环

- 信息溯源：每条分析结论标注数据来源，支持一键跳转原始链接

- 可观测性：完整的日志与 Trace 系统，每个 Agent 的决策过程可追溯

- 报告自动生成：输出标准化 SWOT 分析报告与竞品对比矩阵

### 端到端使用流程

1. 用户在前端界面输入竞品分析需求（目标产品、竞品列表、分析维度）

2. 系统自动分配任务至信息采集 Agent，从公开渠道抓取竞品信息

3. 采集数据经过结构化处理，符合预定义 Schema 后传递至分析师 Agent

4. 分析师 Agent 执行多维度对比分析，生成 SWOT 矩阵与功能对比表

5. 质检 Agent 对分析结果进行事实校验与完整性审查，不合格则打回重做

6. 报告撰写 Agent 将通过校验的分析结果整合为结构化竞品报告

7. 用户在前端查看完整报告，支持溯源跳转、人工修正、Agent 决策回放

## 四、技术说明

### 系统架构图

（建议在此处插入系统架构图，展示前端、后端、Agent 编排层、模型层、数据层及外部服务的调用关系）

### 核心技术栈

| 层级 | 技术选型 |
| --- | --- |
| 前端 | React + TypeScript + Ant Design + ECharts（数据可视化） |
| 后端 | Python FastAPI + Celery（异步任务队列）+ Redis（缓存与消息队列） |
| Agent 编排 | LangGraph / CrewAI，DAG 式任务编排，支持条件分支与循环 |
| 大模型 | Doubao-Seed-2.0-lite（主推理）+ 自定义 Prompt 模板 |
| 数据库 | PostgreSQL（结构化数据）+ ChromaDB（向量存储） |
| 部署 | Docker Compose + Nginx 反向代理 |
| 可观测 | LangSmith / 自建 Trace 系统，记录每次 Agent 调用链 |

### 大模型 / AI 能力使用说明

#### 模型调用

- 主模型：Doubao-Seed-2.0-lite

- 调用方式：方舟 API

- 用途：信息理解、竞品对比、SWOT 生成、报告撰写

#### Agent 设计

- 编排框架：LangGraph DAG

- 通信协议：Function Calling + JSON Schema

- 反馈机制：质检 Agent 自动打回与重试

### 关键工程难点与解决方案

| 难点 | 解决方案 |
| --- | --- |
| Agent 间上下文传递与状态管理 | 基于 LangGraph State 机制，定义全局共享状态与局部 Agent 状态，通过 Checkpoint 实现断点续传 |
| 长文本信息的结构化抽取 | 设计分层 Schema + 分块处理策略，先粗粒度分类再细粒度提取，辅以自一致性校验 |
| 反馈闭环的无限循环风险 | 设置最大迭代次数（3次）+ 渐进降级策略，超限后输出当前最优结果并标注置信度 |

### 部署与访问说明

1. 项目部署于云服务器，通过 Docker Compose 一键启动全部服务

2. 评委可通过 Demo 链接直接访问前端界面

3. 本地运行：克隆仓库后执行 docker-compose up -d 即可启动
