# 开题材料

## 1. 业务介绍

集团信息系统部（Corporate Information System）负责字节跳动信息系统建设。通过构建领先的信息系统，助力公司业务和组织的长期高效经营与发展。与此同时，在安全、隐私、合规层面持续治理，为公司的稳健发展保驾护航。字节跳动的业务模式复杂，用户规模大。到现在，集团信息系统部已经覆盖人事、财务、法务、采购、审批、职场等多个领域，隐私安全工作同步开展，AIGC 创新孵化也在逐步落地实现。

## 2. 课题介绍

### 课题名称

AI 驱动的竞品分析 Agent 协作系统

### 课题背景

企业产品团队在进行竞品分析时，通常需要经历“信息搜集 → 功能对比 → 用户评价整理 → SWOT 分析 → 结构化报告输出”等多个环节，流程重复性高、信息源分散，且对分析人员的行业认知要求较高。本课题实现一个多 Agent 协作的竞品分析系统，模拟一个“数字调研小组”，由 3-4 个专职 Agent 自动完成从公开信息采集到结构化竞品报告的全链路产出，并通过 Agent 间的交叉审查与反馈机制实现自我校验。

### 核心功能

- **角色 Agent**：支持信息采集 Agent（包括问卷设计，问卷调研，用户访谈等）、分析师 Agent、报告撰写 Agent、质检 Agent 等角色，各自负责数据抓取、结构化整理、报告生成、事实校验等独立任务；
- **知识结构化**：定义竞品知识 Schema（功能树、定价模型、用户画像），Agent 产出必须符合 Schema，确保结构化输出一致性；
- **协作与反馈闭环**：Agent 之间通过结构化消息传递（类似 function calling）进行协作，质检 Agent 可将不足打回给采集 Agent，形成 DAG 式任务流转与迭代闭环；
- **信息溯源**：每条分析结论必须标注数据来源，支持 traceability；
- **可观测**：日志可查看，每个 Agent 的决策过程与中间产物均可追溯。

### 考察要点

多 Agent 编排（LangGraph / CrewAI）、结构化知识抽取与 Schema 设计、Agent 间通信协议设计、信息溯源与可追溯性、反馈闭环机制，以及 TRAE 等 AI 编程工具的深度使用。

### 预计周期

3 周：架构设计 0.5 周 + Agent 单体开发 1 周 + 联调与反馈闭环 1 周 + 答辩准备 0.5 周。

### 课题涉及的 AI 工具

主要使用 LangGraph 或 CrewAI 进行多 Agent 编排，LLM 负责信息理解、竞品对比分析、SWOT 生成与报告撰写。目前可能还需要用到 TRAE 设计 Agent 工作流。

## 4. 判断标准

| 评分维度 | 考察要点 | 权重 |
|---|---|---|
| 多 Agent 协作与输出可信度 | - 角色划分清晰，多个专职 Agent（采集 / 分析 / 撰写 / 质检），职责边界明确无重叠<br>- 编排框架（LangGraph / CrewAI）使用合理，DAG 任务流转可视化、可追溯<br>- Agent 间采用结构化消息传递（function calling / 标准 Schema），非纯自然语言对话<br>- 反馈闭环真实可触发：质检 Agent 能识别问题并打回采集 / 分析 Agent 重做，且重做后输出有改善（非伪闭环）<br>- 输出严格符合预定义竞品知识 Schema（功能树、定价模型、用户画像），字段完整、格式一致<br>- 信息溯源完整：每条分析结论可定位到原始数据源（URL / 文档 / 访谈记录），支持一键跳转或溯源查看 | 35% |
| 技术深度与工程完整度 | - 端到端链路完整：数据采集 → Agent 编排 → 知识存储 → 后端接口 → 前端交互，可支持现场演示<br>- 可观测性达标：每个 Agent 的 Prompt、输入、输出、决策过程、Token 消耗均有日志 / Trace 可查<br>- 上下文管理、错误恢复、幻觉抑制有明确策略（如自一致性校验、引用强制、超长上下文分片）<br>- 系统稳定性：异常处理、超时重试、降级机制完备，演示过程无明显卡顿或崩溃<br>- 技术方案有独特或前瞻性思考（如自适应任务拆分、Agent 自评估、动态 Schema 演化） | 25% |
| 业务价值与产品体验 | - 相比传统人工竞品分析，在效率（时间）、覆盖度（信息源）、一致性（结构化）上有可量化的提升<br>- 产品形态贴合企业竞品分析真实工作流，具备可落地性与可扩展性（可换行业、可换竞品对象）<br>- 交互设计流畅：报告查看、溯源跳转、人工介入修正、Agent 决策回放等核心动作易用直观<br>- 设计了清晰的业务闭环（含关键指标如准确率、覆盖率、人工修正率），支持后续运营迭代 | 20% |
| 代码质量与文档 | - 代码风格规范、模块化清晰、关键逻辑注释充分、可读性高<br>- 项目文档齐全：README、架构图、Agent 角色与协议文档、部署说明<br>- Git 提交记录规范，分支管理清晰<br>- TRAE 等 AI 编程工具的使用痕迹清晰，体现深度协作 | 10% |
| 合规、材料与答辩 | - 信息采集合规：遵守目标站点 robots.txt 与服务条款，对外部数据来源有明确授权或公开声明<br>- 数据隐私与安全：用户访谈、问卷数据脱敏处理，无敏感信息泄露<br>- 工具、模型、数据的使用符合公司及挑战赛“工具与资源使用规范”<br>- 提交材料完整：方案文档、演示视频、代码库齐全规范<br>- 答辩讲解清晰有条理，演示直观、问答应对得当 | 10% |
