核心亮点 3：幻觉抑制体系 — 三层防护

第 1 层：引用强制

CitationVerifier 验证每条 [E:N] 引用是否真实存在。Gate-8 阻止引用不存在 evidence 的报告导出。定价 sanitizer 移除无 evidence 支撑的具体价格。

第 2 层：内部一致性校验

8 个一致性 gate 检测报告内部矛盾：定价一致性（verified=无时不出现价格）、覆盖率数字一致性、JSON 防泄漏、引用编号有效性等。

第 3 层：超长上下文分片

当 evidence 超过 LLM context window 时，自动按 product 分组、按 token 估计分批处理，确保不会因信息截断导致幻觉。（P0-5 已实现）

答辩话术

"我们的幻觉抑制包含三个层次：引用强制确保每条结论绑定真实 evidence；内部一致性校验检测报告内部矛盾；超长上下文分片防止因信息截断导致的幻觉。任何一个 gate 失败都会阻止报告导出。"

答辩 Q&A 准备

Q1: 幻觉抑制具体怎么做？
引用强制（CitationVerifier+Gate-8）+ 内部一致性校验（8个gate）+ 超长上下文分片（P0-5已实现）。任何一个gate失败都会阻止报告导出。
Q2: Feedback loop 是真的吗？
是真的。Reviewer 发现evidence不符合要求时，自动将claim降级为rework_required，生成补证任务，Rework后重新进入Reviewer。数据库里真实写入了Before/After对比数据。
Q3: signed claims = 0 是不是失败？
恰恰相反。signed claims = 0说明系统正确识别了证据不足，没有把不符合要求的evidence签成正式claim。这比生成有幻觉的完整报告更符合比赛要求。
Q4: 超长上下文怎么处理？
系统实现了evidence分片机制。当evidence超过LLM context window时，自动按product分组、按token估计分批处理，确保不会因信息截断导致幻觉。
Q5: 3个Schema怎么对齐？
报告严格按照功能树、定价模型、用户画像三个Schema组织。每个claim都绑定到对应Schema的维度，Evidence Appendix中包含schema_key字段用于追溯。
Q6: 相比传统方法有什么优势？
时间：3人天→18分钟（24x）；覆盖：5-8个→50+个（8x）；可信度：从主观判断到evidence-binding+gate；可追溯：每条结论都有URL溯源。
