## M4：可信链路加固（6 月 3 日）

**目标**：从来源侧和报告侧双重保障可信生成

**完成内容**：

**Evidence Contract 硬门控**（`deep_report.py`）：
- `DIMENSION_SOURCE_REQUIREMENTS` 配置表：每个分析维度设定来源白名单
  - `pricing_model` / `pricing`：拒绝 `third_party_article`、`social_media`
  - `sla` / `compliance` / `security`：拒绝第三方和 blog
  - 所有维度默认拒绝 `third_party_article`
- 在 `run_deep_report_workflow` 入口对所有 evidence 执行 gate

**Writer Agent blocked 降级**（`writer.py`）：
- `is_blocked` 模式下禁止"最优"/"top pick"/"🥇"等强推荐语言
- 评分卡降级为"暂无法评分"，POC/TCO 降级为"待核验"

**URL Fetch 实现**（`multi_round_search_service.py`）：
- 添加 `fetch_source_candidates()` 方法，真正抓取 snippet_only URL
- `_extract_passages_from_text()` 从抓取内容抽取证据段落
- `_determine_trust_tier()` 基于域名判断信任等级
- **效果显著**：证据数 58→380，可用证据 0→255，A/B 级占比从 0% 提升到 76%

**E2E 验证**（Run ID `run_fd7ec6196a594fc4`，39 evidence，14 claims）：
- 13 节并行生成（3 workers），耗时 1073 秒（18 分钟）
- `report_status = blocked_consistency` ✅
- Pricing model 维度的 `web_page` 来源 evidence 被正确标记 `gate_rejection` ✅
- 评分卡正确降级 ✅

## M6：上下文分片（6 月 7 日）

**目标**：解决超长 evidence 列表导致的上下文截断和幻觉

**完成内容**：
- `_estimate_evidence_tokens()`：基于字符数估算 token 数（约 2 chars/token）
- `_chunk_evidence_for_llm()`：按 product 分组 evidence，max_tokens 超出时分块
- `_call_llm_with_evidence_chunks()`：自动分批调用 LLM，多 chunk 时合并结果
- 在 `_llm_generate_table` 内部集成 chunking

**其他修复**：
- 证据附录改为折叠卡片（`<details>/<summary>`），前 5 条默认展开，unsigned 标记清除
- API 自动返回最新版本报告（新增 `_find_latest_md_path()`）
